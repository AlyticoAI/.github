## Type of change
- [ ] `feature` — new functionality
- [ ] `fix` — non-urgent bug fix
- [ ] `hotfix` — urgent production fix
- [ ] `chore` / `docs` — no product behavior change

## Problem statement
_What is broken, missing, or needed? Who is affected and how? Link the issue/ticket if one exists._

## Solution approach
_What does this PR actually do, at a high level?_

## Options explored
_What alternatives did you consider (including "do nothing")? Bullet list is fine._

## Why this option
_Why this approach over the alternatives above — performance, simplicity, risk, time, consistency with existing patterns, etc._

## Change explanation
_Walk a reviewer through the diff: which files changed and why, in the order that makes it easiest to review. Call out anything non-obvious._

## Testing strategy
_How was this verified? Unit/integration tests added or updated, manual test steps, edge cases considered._

## Test result evidence
_Paste test output, screenshots, screen recordings, or preview links. "Works on my machine" is not evidence._

## Risk & rollback
- Does this touch a path listed as sensitive in [CODEOWNERS](.github/CODEOWNERS)? _(Code Owner review will be auto-requested if so.)_
- Breaking change? Y/N — if Y, what breaks and who needs to know?
- Rollback plan if this needs to be reverted after merge:

## Checklist
- [ ] Self-reviewed the diff before requesting review
- [ ] No secrets, credentials, or `.env` values committed
- [ ] Tests added/updated and passing locally
- [ ] No unresolved merge conflicts with target branch
- [ ] Related issue/ticket linked above
